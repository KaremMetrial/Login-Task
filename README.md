# Dynamic Login Challenge – Laravel

## فكرة النظام
كلمة المرور لا تكون ثابتة أو عشوائية، بل تعتمد على التاريخ الحالي.

## القاعدة المنطقية
Password = (Day × Month) + Constant

مثال:
2 فبراير → (2 × 2) + 10 = 14

## لماذا هذا الأسلوب؟
- بسيط ويمكن حسابه ذهنياً
- لا يعتمد على التخزين
- يتغير مع الزمن
- قابل للتنبؤ لمن يفهم القاعدة

## لماذا ليس عشوائياً؟
لأن الناتج يعتمد على قيم ثابتة (اليوم والشهر) ومعادلة رياضية واضحة.

## الميزة الإضافية
تم منع إعادة استخدام نفس كلمة المرور مرتين متتاليتين باستخدام Session.
